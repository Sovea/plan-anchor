# Local-Fix Loop Guard

Trigger this checklist after two consecutive attempts mainly fix secondary failures without completing or materially advancing the active Work Unit.

## Reassess

- [ ] Restate the active Work Unit.
- [ ] State the acceptance criterion it serves.
- [ ] Identify the blocker preventing completion.
- [ ] Classify the blocker as incidental, structural, requirement gap, or environmental.
- [ ] Identify whether the current fixes still advance the mission.

## Decide

Choose exactly one next action:

- [ ] Continue with one bounded fix that directly completes the Work Unit.
- [ ] Replan because the blocker is structural.
- [ ] Ask the user because a requirement or tradeoff is unresolved.
- [ ] Produce Handoff State because progress is blocked or should pause.

## Constraint

Do not keep patching unrelated errors without a renewed mainline hypothesis.
