# Recovery Checklist

Use this checklist when resuming from Handoff State or a durable anchor.

- [ ] Ask for Handoff State or durable anchor path if neither is provided.
- [ ] Restate the original mission.
- [ ] Restore acceptance criteria status.
- [ ] Restore completed Work Units with evidence.
- [ ] Identify the active or next Work Unit.
- [ ] Review known blockers, risks, verification gaps, and drift.
- [ ] Check current branch and working tree status when relevant.
- [ ] Check whether current repository state still matches the handoff assumptions.
- [ ] Trust repository state over stale handoff or durable anchor content if they conflict.
- [ ] Update the Progress Ledger if current state differs from the handoff.
- [ ] If durable state is enabled, update the durable anchor after recovery.
- [ ] Continue with the smallest next action that advances the active Work Unit.
