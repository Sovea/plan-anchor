# Drift Check

Compare current implementation against the Mission Contract and Execution Plan.

- [ ] Mission remains unchanged.
- [ ] Scope has not expanded silently.
- [ ] Non-goals remain excluded.
- [ ] Acceptance criteria have not been dropped or weakened.
- [ ] Architecture and API decisions still match the approved plan.
- [ ] Verification strategy remains valid.
- [ ] Any deviation is recorded with reason, impact, and approval status.

Stop and ask the user when drift changes user-visible behavior, architecture, scope, risk, or acceptance criteria.
